Therm-Pro API

← Dashboard

Data

POST /api/data
Submit probe temperature readings. The ESP32 calls this endpoint.
{
  "probes": [
    {"id": 1, "temp_f": 250.0},
    {"id": 2, "temp_f": 165.3},
    {"id": 3, "temp_f": 180.1},
    {"id": 4, "temp_f": -999.0}
  ],
  "battery": 85
}
A temp_f of -999.0 indicates a disconnected probe.

Session

GET /api/session
Returns the current cook session with all probes, history, and alert configuration.
{
  "probes": [
    {"id": 1, "label": "Pit", "current_temp": 250.0, "connected": true, "alert": {"low_temp": 225, "high_temp": 275}},
    {"id": 2, "label": "Brisket", "current_temp": 165.3, "connected": true, "alert": {"target_temp": 203}}
  ],
  "history": [{"timestamp": "...", "temps": [250, 165, 180, 190], "battery": 85}],
  "started": "2026-03-28T12:00:00Z"
}
POST /api/session/reset
Clear all temperature history. Probe labels and alert configs are preserved. Call before each cook.

Alerts

POST /api/alerts
Set alert thresholds for a probe. All fields in the alert object are optional—omit or set to null to clear.
// Target temp (meat probe)
{"probe_id": 2, "alert": {"target_temp": 203.0}}

// Range alert (pit probe)
{"probe_id": 1, "alert": {"low_temp": 225.0, "high_temp": 275.0}}
Alerts fire once, reset after 3°F hysteresis, rate-limited to 60s minimum.

WebSocket

GET /api/ws
WebSocket endpoint for live temperature updates. Server pushes a JSON reading on each new data point.
// Each message:
{"timestamp": "...", "temps": [250, 165, 180, 190], "battery": 85}

Firmware OTA

GET /api/firmware/latest
Returns the latest firmware version info. Returns 404 if no firmware has been uploaded.
{"version": 2, "download_url": "/api/firmware/download"}
GET /api/firmware/download
Download the latest firmware binary (.bin file).
POST /api/firmware/upload
Upload a new firmware binary. Multipart form with firmware (file) and version (integer) fields.
curl -X POST http://localhost:8088/api/firmware/upload \
  -F "firmware=@esp32/.pio/build/esp32/firmware.bin" \
  -F "version=2"

Health

GET /healthz
Returns "ok" if the server is running.
